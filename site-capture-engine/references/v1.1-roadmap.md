# site-capture-engine v1.1 — credential-gated additions roadmap

Each item below ships as an **empty no-op hook** in v2.0's `capture_site.py`. v1.1 fills the hook
bodies once credentials are wired in tier-3. No rework — just credential gates.

## Hooks

### 1. Lighthouse / PageSpeed snapshot per page

| Field | Value |
|---|---|
| Hook function | `hook_lighthouse(url, out_dir)` |
| Insertion point | `capture_site.py` Phase 3, per-page loop (after meta extraction) |
| Credential prerequisite | Lighthouse CLI installed locally OR PageSpeed Insights API key in tier-3 |
| Output | `lighthouse/<slug>-lighthouse.json` (performance, accessibility, best-practices, SEO scores + diagnostics) |
| Regression test | Run on EV homepage; confirm JSON has all 4 category scores + ≥1 diagnostic per category |
| When to fill | Lighthouse CLI available on host (free, no API key needed — just `npm install -g lighthouse`) |
| Build estimate | ~1 hour |

### 2. axe-core accessibility audit per page

| Field | Value |
|---|---|
| Hook function | `hook_axe_accessibility(url, out_dir)` |
| Insertion point | `capture_site.py` Phase 3, per-page loop (after meta extraction) |
| Credential prerequisite | Playwright + `@axe-core/playwright` npm package |
| Output | `accessibility/<slug>-axe-audit.json` (violations, incomplete, passes counts + per-violation details) |
| Regression test | Run on EV homepage; confirm JSON has violations array + each violation has `id`, `impact`, `nodes[]` |
| When to fill | Playwright installed (already a prerequisite) + axe-core package added |
| Build estimate | ~1 hour |

### 3. GA4 historical traffic snapshot per URL

| Field | Value |
|---|---|
| Hook function | `hook_ga4_traffic(url, out_dir)` |
| Insertion point | `capture_site.py` Phase 3, per-page loop |
| Credential prerequisite | GA4 service account JSON in tier-3 + GA4 property ID per client |
| Output | `analytics/ga4-traffic-<slug>.json` (sessions, pageviews, bounce rate, avg duration — 90-day window) |
| Regression test | Run on EV property; confirm JSON has ≥1 URL with non-zero sessions |
| When to fill | GA4 credentials wired in tier-3 (Phase 4 of client-onboarding-automation) |
| Build estimate | ~2 hours (Google Analytics Data API v1 integration) |

### 4. GSC indexed-pages snapshot

| Field | Value |
|---|---|
| Hook function | `hook_gsc_indexed_pages(domain, out_dir)` |
| Insertion point | `capture_site.py` Phase 5, domain-level (after DNS/WHOIS/SSL) |
| Credential prerequisite | GSC service account JSON in tier-3 + verified property per client |
| Output | `gsc/gsc-indexed-pages.json` (indexed count, coverage issues, sitemaps status) |
| Regression test | Run on EV property; confirm JSON has indexed URL count + ≥1 sitemap entry |
| When to fill | GSC credentials wired in tier-3 (Phase 4 of client-onboarding-automation) |
| Build estimate | ~1.5 hours (Search Console API integration) |

### 5. GBP listing snapshot per location

| Field | Value |
|---|---|
| Hook function | `hook_gbp_listing(domain, out_dir)` |
| Insertion point | `capture_site.py` Phase 5, domain-level |
| Credential prerequisite | GBP API credentials (OAuth2 service account) + multi-location loop |
| Output | `gbp/gbp-listing-<location-id>.json` (name, address, phone, categories, hours, attributes, photos count, review count + avg rating) |
| Regression test | Run on EV; confirm JSON has ≥1 location with name, address, primary category |
| When to fill | GBP API access approved (application pending per [G6] GBP API toolbuild) |
| Build estimate | ~2 hours (GBP API v4 integration + multi-location loop) |

### 6. DataForSEO backlink-profile snapshot

| Field | Value |
|---|---|
| Hook function | `hook_dataforseo_backlinks(domain, out_dir)` |
| Insertion point | `capture_site.py` Phase 5, domain-level |
| Credential prerequisite | DataForSEO Backlinks subscription active (separate from Labs) |
| Output | `data/dfs-backlinks-profile.json` (referring domains count, backlinks count, top referring domains, anchor text distribution) |
| Regression test | Run on AJ Long domain; confirm JSON has referring_domains > 0 + top_domains array |
| When to fill | DataForSEO Backlinks subscription activated (paid, ~$50/mo) |
| Build estimate | ~1 hour (endpoint already documented in `references/dataforseo-endpoints.md`) |

### 7. Cookie inventory

| Field | Value |
|---|---|
| Hook function | `hook_cookie_inventory(url, out_dir)` |
| Insertion point | `capture_site.py` Phase 3, per-page loop |
| Credential prerequisite | Playwright live browser session (already a prerequisite for screenshots) |
| Output | `cookies/<slug>-cookies.json` (per-cookie: name, domain, path, expires, httpOnly, secure, sameSite, value length) |
| Regression test | Run on EV homepage; confirm JSON has ≥1 cookie with name + domain fields |
| When to fill | Playwright installed (already a prerequisite) — just needs the cookie extraction code |
| Build estimate | ~1 hour |

| GBP API OAuth2 (tier-3) | 5 | [G6] GBP API toolbuild approval |
| DataForSEO Backlinks (paid) | 6 | Subscription activation (~$50/mo) |
| Playwright (already required) | 7 | Already installed for screenshots |
